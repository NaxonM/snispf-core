# Changelog

All notable changes to this project are documented in this file.

## [v0.1.6] - 2026-04-18

### Fixed
- PowerShell release matrix packaging no longer collides with the read-only automatic `$IsLinux` variable in `build_release_matrix.ps1`.
- Bundle layout initialization now uses an explicit Linux service file flag, restoring successful end-to-end bundle generation on Windows hosts.

## [v0.1.5] - 2026-04-18

### Added
- Linux bundles now include a `systemd` service template (`snispf.service`) and service installer/manager script (`install_linux_service.sh`).

### Changed
- Windows/Linux release bundles now use a generated default config (`--generate-config`) to keep runtime defaults aligned with the current core behavior.

### Documentation
- Added recommended Linux service deployment guidance, including status/restart/log commands and default `LimitNOFILE` behavior.

## [v0.1.4] - 2026-04-18

### Added
- OpenWrt matrix now includes an x86_64 target and output binary.
- OpenWrt build now generates per-architecture `*_bundle.tar.gz` archives that include:
  - architecture-specific binary
  - `openwrt_snispf.sh` deployment script
  - generated default `config.json`

### Changed
- OpenWrt default bundle config is generated by core and normalized to non-failover defaults (`LOAD_BALANCE=round_robin`, `AUTO_FAILOVER=false`, `FAILOVER_RETRIES=0`).
- Release publishing order was updated to group Windows/Linux core artifacts together and OpenWrt artifacts together.

### Fixed
- OpenWrt init script now sets `procd` nofile limits, reducing `socket: too many open files` failures under sustained load.
- OpenWrt helper script handling in build scripts now fails fast when required script input is missing.

### Documentation
- OpenWrt bundles and deployment script are now documented as the easiest deployment path.

## [v0.1.3] - 2026-04-18

### Added
- Config precedence warnings at startup when ENDPOINTS[0] conflicts with top-level CONNECT_IP, CONNECT_PORT, or FAKE_SNI.

### Changed
- Improved strict wrong_seq reliability with clearer connection-drop warnings and stronger failover attempt handling.
- Enhanced Linux raw injector route awareness and send-path stability after interface/route changes.
- Improved OpenWrt operational recovery with route-aware source-IP dialing and faster watchdog restart behavior.
- OpenWrt installer flow now supports delayed post-install restart and guided watchdog installation.

### Fixed
- wrong_seq runtime diagnostics now surface more failure modes in logs.
- OpenWrt script examples now consistently use ash invocation style.
- CLI --info behavior is now clearly documented as config-independent.

### Documentation
- Synced docs with runtime defaults and behavior:
  - ENDPOINTS precedence model and warnings.
  - round_robin default load-balancing.
  - AUTO_FAILOVER default behavior.
  - OpenWrt watchdog guidance and strict wrong_seq multi-WAN notes.
